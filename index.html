Overview

What AICA-Bench provides

The repository is already structured as a practical benchmark toolkit. It contains benchmark YAMLs, CLI entrypoints, model interface modules, instruction builders, preprocessing examples, and test files.

Dataset Building

Construct random or balanced benchmark subsets from emotion-related datasets using the repository CLI.

Instruction Generation

Produce standard or CoT-style instructions for understanding, reasoning, and generation tasks.

Unified Evaluation

Run model adaptation pipelines and compute benchmark metrics through a consistent command workflow.

Representative datasets

AffectNet, EmoSet, Emotic, Emotion6, FI, FlickrLDL, TwitterLDL, ArtPhoto, Abstract, and FindingEmo are already reflected in example configs or preprocessing scripts.

Workflow

From raw data to benchmark run

Step 1

Build datasets

aica-vlm build-dataset run benchmark/example.yaml --mode random
Step 2

Generate instructions

aica-vlm build-instruction run examples/config-yaml/instructions-builder/EU_observer_emotion_emoset.yaml
aica-vlm build-instruction run-cot examples/config-yaml/instructions-builder/EU_observer_emotion_emoset_CoT.yaml
Step 3

Run benchmark

aica-vlm benchmark benchmark/example.yaml

Resources

Repository structure

Core package

  • src/aica_vlm contains the benchmark package.
  • adaptation/ includes model interfaces and runners.
  • instructions/ provides prompt-building modules.
  • metrics/ contains metric implementations.

Examples and tests

  • benchmark/ includes a starter benchmark YAML.
  • examples/ contains configs and preprocessing examples.
  • test/ includes repository test coverage.
  • Third-party datasets may require separate access and license compliance.
