Overview

A repository-first benchmark page with room for paper artifacts

The current public release is centered on repository materials that already exist. AICA-Bench exposes practical benchmark workflows today, while the paper-specific assets are staged as upcoming modules instead of being fabricated.

Abstract

What the benchmark already covers

AICA-Bench is built to evaluate vision-language models on affective image content analysis. The repository already unifies benchmark dataset construction, instruction generation, model adaptation, and evaluation into one reproducible workflow.

The public code release focuses on practical usage: benchmark YAMLs, CLI entrypoints, prompt builders, model interface modules, preprocessing examples, and test files. The project page now reflects that structure in a format that reads like an academic benchmark homepage instead of a generic repository landing page.

Highlights
Dataset building Construct random or balanced benchmark subsets from emotion-related datasets.
Instruction generation Produce standard and chain-of-thought instructions for multiple affective tasks.
Model adaptation Run open-source and configurable evaluation pipelines from one repository.
Unified metrics Keep evaluation logic and task organization inside the same benchmark package.

Benchmark

Three affective task families, one benchmark workflow

01

Emotion Understanding

Evaluate whether models can identify and interpret emotional cues from visual content.

02

Emotion Reasoning

Measure whether models can justify affective conclusions and reason about emotional evidence.

03

Emotion-Guided Generation

Assess whether generated outputs follow explicit emotion-conditioned instructions and goals.

Workflow

From raw benchmark inputs to runnable evaluation

Step 1

Build datasets

Generate benchmark subsets from benchmark YAML configuration files.

aica-vlm build-dataset run benchmark/example.yaml --mode random
Step 2

Generate instructions

Build standard or chain-of-thought prompts for affective tasks.

aica-vlm build-instruction run examples/config-yaml/instructions-builder/EU_observer_emotion_emoset.yaml
aica-vlm build-instruction run-cot examples/config-yaml/instructions-builder/EU_observer_emotion_emoset_CoT.yaml
Step 3

Run benchmark

Execute benchmark evaluation with the repository-provided CLI.

aica-vlm benchmark benchmark/example.yaml

Repository structure

Core implementation areas

  • src/aica_vlm contains the benchmark package.
  • adaptation/ includes model interfaces and runners.
  • instructions/ provides prompt-building modules.
  • metrics/ contains evaluation implementations.
Examples and compliance

Public configs, private data sources

  • examples/ contains preprocessing examples and benchmark templates.
  • test/ provides repository test coverage.
  • Third-party datasets remain subject to their original licenses and access conditions.
  • Dialogue instruction sets and paper assets can be added later as dedicated releases.
